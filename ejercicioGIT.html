    6. ¿Cómo puedo gener un tag?

      Escribiendo el comando git tag. En Git, un "tag" (etiqueta) es una manera de asignar un nombre descriptivo y permanente a un punto específico en la historia del repositorio. Cuando creas un tag en Git, estás creando una referencia fácilmente reconocible y accesible a un punto específico en la historia de tu proyecto.

    8. ¿Cómo puedo moverme a un commit especifico?

      Para cambiar de una rama a otra usaré el comando git checkout [nombre-rama] Hay que tener en cuenta que hacer un commit en Git es hacer un registro de cambios en el repositorio. Cada commit en Git tiene asociado un identificador único (SHA-1 hash) que lo distingue de otros commits. Cuando realizas un commit en Git, estás guardando los cambios que has hecho en tus archivos en ese momento específico. Esto incluye agregar nuevos archivos, modificar archivos existentes o eliminar archivos. Además, cada commit generalmente incluye un mensaje descriptivo que resume los cambios realizados en ese commit, lo que ayuda a otros colaboradores a entender qué se ha modificado y por qué. ======= Con el comando git merge podré fusionar ramas de mi proyecto.
