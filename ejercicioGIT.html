  1. ¿Que es un sistema de control de versiones?

    Un sistema de control de versiones (VCS, por sus siglas en inglés, Version Control System) es una herramienta que ayuda a los equipos de desarrollo de software a gestionar los cambios en el código fuente a lo largo del tiempo.

  2. ¿Cual es la diferencia entre git y github?

    Git es un sistema de control de versiones distribuido de código abierto diseñado para manejar todo, desde proyectos pequeños hasta muy grandes, con rapidez y eficiencia. Fue creado por Linus Torvalds en 2005 para el desarrollo del kernel de Linux. Git permite a los desarrolladores llevar un registro de los cambios en su código, colaborar con otros desarrolladores de manera eficiente, y mantener un historial completo y detallado de todos los cambios realizados en el código de un proyecto. Mientras GitHub es una plataforma de desarrollo colaborativo basada en la web que utiliza Git para control de versiones. Facilita la colaboración entre desarrolladores permitiendo almacenar repositorios de Git en la nube, revisar código, gestionar proyectos y construir software junto a millones de desarrolladores. En resumen, Git es el sistema de control de versiones, mientras que GitHub es una plataforma que utiliza Git para alojar repositorios y facilitar la colaboración en proyectos de desarrollo de software.

  3. ¿Cómo creo una rama en git?

    Con el comando git branch {nombre de la rama}

  4. ¿Cómo clono un repositorio?

    Con el comando git clone [URL del repositorio]

  5. ¿Cómo puedo bajar los ultimos cambios de un repositorio?

    Con git pull [alias] [rama]: Actualiza tu repositorio local con los cambios desde el repositorio remoto.

  8. ¿Cómo puedo moverme a un commit especifico?

    Para cambiar de una rama a otra usaré el comando git checkout [nombre-rama]

  10. ¿Cómo puedo borrar un tag?

  11. ¿Cómo puedo ver los cambios que estan por enviarse al repositorio?

    Con git diff --staged Aunque "git show" también nos mostrará los cambios que han existido sobre un archivo.

  12. ¿Para que sirve el comando git remote -v?

    Al ejecutar git remote -v, obtendrás una lista de todos los repositorios remotos que están configurados para tu repositorio local, así como las URLs asociadas a cada uno de ellos.

    Usando git config --global user.name "Tu Nombre", para saber mi nombre Y usando y $ git config --global user.email johndoe@example.com, para configurar mi correo
